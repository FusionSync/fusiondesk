## Do Not Misread The Current Code

The current core code is a foundation, not the whole runtime.

Rules:

```text
Do not let modules open sockets directly.
Do not let modules call app window classes.
Do not let core include Qt.
Do not let legacy Source headers enter core.
Do not bypass SessionManager for dynamic feature enable/disable.
Do not treat PacketEnvelope protocolMajor/protocolMinor as a module payload schema version.
Do not put FDIN, FDCL, raw-frame, FDPP, FDRT, or future module payload compatibility decisions in RuntimeHost, SessionMainline, ModuleHost, ChannelAllowlistValidator, or ProtocolValidator.
Do not put new headers in include/fusiondesk/module, network, policy, protocol, or session.
Do not put new sources in src/module, network, or policy.
```

Use only these roots for new code:

```text
include/fusiondesk/core/...
include/fusiondesk/runtime/...
include/fusiondesk/modules/...
include/fusiondesk/platform/...
include/fusiondesk/adapters/...
include/fusiondesk/bindings/...

src/core/...
src/runtime/...
src/modules/...
src/platform/...
src/adapters/...
src/bindings/...
```

If compatibility forwarding headers are needed later, they must be explicit and documented. Empty compatibility roots must not become active ownership roots.

## Current Architecture Gaps

The gaps to close before broad rebuild:

```text
PacketEnvelope v1, RequestTracker, ProtocolValidator, PacketCodec, CapabilityPayloadCodec, CapabilityNegotiator, CapabilityExchange, ChannelAllowlistValidator, ChannelRegistry, PriorityScheduler, default MVP channel specs, NetworkManager skeleton, NetworkManager channel rebind boundary, NetworkManager transport boundary, ITransportSocket, SocketGroup, DiagnosticsSink, SessionManager, ClientSession, AgentSession, and RuntimeHost exist. Full protocol lifecycle still needs runtime/session storage of negotiated capabilities and remote module declarations, while module payload schema validation remains module-owned.
ProtocolValidator consumes maxPayloadBytes as a configured validation option; CapabilityExchange can derive that option from negotiated capabilities, but runtime/session storage of negotiation results is not implemented yet.
ChannelAllowlistValidator checks negotiated channel types, packet types, and message kinds; ChannelRegistry owns channel specs, channel ids, readiness, pressure, degraded state, and binding. SessionCreateOptions can carry ProductProfile minimum channel specs into NetworkManager at session creation. This registers channel specs only; concrete transport binding and markReady still come from the transport/channel adapter. NetworkManager can rebind a logical channel in tests while preserving router subscriptions. SocketGroup is integrated into NetworkManager for packet-byte writes by ChannelSpec.socketClass. QtTcpTransportSocket exists as the first optional Qt transport adapter under `adapters/qt`; accepted sockets are detached from their Qt parent before adapter ownership transfer. QtPacketChannel can bind a Qt transport socket into the NetworkRouter channel model and decode incoming TCP stream frames through PacketCodec before submitting packets to NetworkRouter. QtChannelBinder can register the Qt transport socket, register or reuse ChannelSpec, bind QtPacketChannel, and mark ChannelRegistry ready. Session/runtime now has a pure C++ peer connection plan resolver under `runtime/connection` that resolves requested ChannelKey entries against known ChannelSpec values, rejects duplicate channel/endpoint plans, and keeps concrete endpoint parsing out of the common contract. It also has a pure C++ reconnect orchestration plan resolver that turns service-selected degraded ChannelKey values into client replacement TCP channels, agent replacement TCP listeners, reason/keyframe intent, and teardown-after-success intent without creating sockets or calling Session. ReconnectCoordinator now provides the service-level pure C++ frame for plan resolution, runtime replacement execution, Session reconnect handoff, and teardown dispatch through executor interfaces, without owning Qt or socket details. runtime/connection now also has a pure C++ reconnect runtime service owner that composes ReconnectCoordinator, RequestTracker, ReconnectTeardownService, and the teardown executor behind one start/run/expire/snapshot boundary. runtime/connection now also has a pure C++ reconnect teardown contract that turns post-rebind teardown intent into correlated old-transport teardown commands, maps those commands to control-channel `PacketEnvelope` requests with target-channel `FDRT` payloads, dispatches requests through `NetworkRouter`, subscribes for interim/terminal responses, tracks them through `RequestTracker`, handles peer-side FDRT requests through `IReconnectTeardownCloseTarget`, expires timed-out requests through the same tracker path, and summarizes terminal Response/Error results while treating Ack as interim only. runtime/tunnel now has a pure C++ `TunnelReconnectExecutor` contract that adapts `ReconnectOrchestrationSidePlan` into tunnel replacement requests for LAN, relay, or direct P2P backends while still implementing the existing `IReconnectReplacementExecutor` boundary. runtime/tunnel now also has `TunnelCandidateProfile` negotiation that selects compatible client/agent candidate pairs per ChannelKey and mode policy, plus `ITunnelTransportFactory` as the concrete LAN/relay/direct adapter factory boundary with factory requests carrying local and peer candidates when built from a selection. QtSessionTransportConnector exists under `runtime/qt`; it can create client-side Qt TCP transports from a transport profile, adopt accepted server-side QTcpSocket instances, drive QtChannelBinder, and prepare reconnect replacement channels without marking them ready. QtRuntimeTransportManager owns QtSessionTransportConnector instances and server-side QTcpServer listeners per RuntimeHost sessionId, can prepare reconnect replacements for a RuntimeHost session from connect profiles or accepted sockets, can drive Session reconnect for those replacements in one runtime-facing API, closes/removes the previous active Qt transport for that ChannelKey after a successful one-call reconnect, and implements the `IReconnectTeardownCloseTarget` adapter used by ReconnectTeardownHandler for peer-side FDRT teardown requests. ## Current Closed Loop
```

The current implemented control loop is:

```text
Session.start
  -> creates NetworkManager
  -> creates PolicyEngine
  -> creates ModuleHost
  -> sees minimum channel specs registered by SessionManager
  -> injects NetworkManager.router into ModuleRuntime
  -> injects NetworkManager.registry into ModuleRuntime

ModuleHost.addModule
  -> rejects duplicate or empty module id
  -> rejects unsupported session role
  -> rejects unsupported local platform
  -> attaches accepted module

ModuleHost.startAllowedModules
  -> checks required module presence
  -> checks required channel readiness through ChannelRegistry
  -> calls IPolicyEngine::authorizeModule
  -> starts only policy-allowed modules
```

RuntimeHost can now explicitly mount ProductProfile required modules into a session through ModuleComposer. DisplayModuleFactory provides the display MVP behavior, InputModuleFactory provides first contract modules for input and accepts adapter dependencies from RuntimeHost, and FeatureModuleFactory provides lifecycle-only skeletons for remaining non-display feature modules. Session.start still creates the ModuleHost but does not automatically consume every ProductProfile module family.

## Rebuild Principle

Legacy code is reference-only. New `FusionDesk` code is implemented by capability, not by directory copy or wrapper inheritance.

Reference-to-new mapping:

```text
old screen capture behavior -> platform/<os>/display or modules/display/agent
old render behavior -> adapters/qt/display or modules/display/client
old TCP channel behavior -> adapters/transport or runtime/qt
old feature manager behavior -> new modules/<feature> implementation
old platform API lessons -> platform/<os>
old app startup behavior -> apps/<product>
```

Forbidden dependency paths:

```text
Source/App private class -> FusionDesk/core
Source/Modules private class -> FusionDesk/core
Source module class -> FusionDesk/modules as the target implementation
Qt socket class -> FusionDesk/core
platform API -> FusionDesk/core
business policy check hidden in feature module
feature module owning a concrete socket
```
