# FusionDesk TODO Tracker

This is the live tracker for the clean `FusionDesk` rebuild. It replaces the old
historical refactor TODO files that were removed from `docs/architecture`.

The rule for the next phase is fast closure by item, followed by stage-level
audit:

```text
one closure item = goal + owned files + focused proof + doc status
implement the item, run its focused proof, record the result, move on
large architecture frame first, then one high-level smoke per new frame
detail tests only when a wire contract, public API, platform adapter, or cross-thread boundary is at risk
do not repeatedly expand one feature before the next bounded item can close
after a batch, run full build, full CTest, purity scans, and code review
minimum ProductProfile start is all-or-nothing; partial attachment start belongs to later module-specific work
H264 is retained as a near-term display codec closure lane; raw Windows-Windows remains the stable fallback until H.264 passes its gate
```

## Current Direction

The current priority is to keep the minimum product mainline stable while
production UI/service owners start consuming the runtime diagnostics snapshots
that the PC shell can already print as stable stdout records. Deeper feature
behavior remains paused until those mainline diagnostics are visible from
product surfaces without parsing shell stderr:

```text
RuntimeHost
  -> SessionManager
  -> link/channel binding report
  -> ChannelRegistry ready state
  -> ProductProfile multi-module mount
  -> ModuleHost start gates
  -> SessionRuntimeDiagnosticsSnapshot
  -> product UI/service reader
```

## Testing Policy

Use tests at the right level:

```text
new architecture frame -> one high-level smoke
wire/protocol shape -> focused contract smoke
Qt or platform adapter -> one adapter smoke
bug or regression -> narrow regression test
mainline module tests -> module id/version matching, startup gate, diagnostics
module payload tests -> module-owned schema, operation, downgrade, and rejection behavior
```

Avoid adding many edge-case tests before the owning frame is stable. Track
known gaps here instead.

## Verification Commands

Minimum local verification for framework changes:

```powershell
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
rg -n "#include <Q|QString|QByteArray|QObject|QTcpSocket|QThread|QVariant|QJson|QWindow|QAndroid" include/fusiondesk/core src/core include/fusiondesk/modules src/modules
rg -n "Source/|ThirdParty/" include/fusiondesk/core src/core include/fusiondesk/modules src/modules
```

The `rg` scans should have no matches.
